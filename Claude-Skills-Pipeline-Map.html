Phase A Coverage

Harvesting Stage

STRONG COVERAGE
95% — 5 skill categories produce intake intelligence

Digital Storefront Audit, GBP Audit, Brand Content, UGC, and Practice Performance skills comprehensively cover intake assessment. The only gap: no automated practice profile data extraction (name, address, staff roster) — that's manual entry or PMS-sourced.

Integrating Stage

GOOD COVERAGE
80% — Strategy, Calendar, Content Engine, Referral Lab

Marketing Strategy provides the storefront blueprint. Social Calendar + Content Engine power the content pipeline. Gap: no skill currently generates the actual storefront template or provider bio cards — that's the Storefront Generation Engine (platform-native).

Testing Stage

IDENTIFIED GAP
40% — Re-audit + Scheduling Scanner available, but no dedicated post-build QC skill

The Storefront Audit can re-run against generated sites, and the Scheduling Link Scanner validates booking links. But there's no dedicated "post-build validation" skill that checks Gen4-specific template compliance, brand asset accuracy, or provider bio correctness. Recommendation: create a Storefront QC skill.

Live / Monitoring Stage

GOOD COVERAGE
75% — Performance + Content + Dept Reporting cover ongoing monitoring

Practice Performance powers executive dashboards. Content skills power ongoing brand monitoring. Department Reporting covers HQ operations. Gap: no skill currently produces H-Score composite calculations or CCA certification tracking — those are platform-native WS3 functions.

WS3: Culture OS

LOWEST SKILL COVERAGE
25% — Only H-Score + Playbook fed by skills; 6 interfaces are platform-native only

Culture OS was designed around aggregate vendor scores (Cinnamon, Swell, Dental Intel), not practice-level content analysis. Morning Huddle, Champion Hub, CCA Academy, Recognition Feed, and Culture Admin Console have no Claude Skill inputs. This is by design — these are interactive platform features, not report-driven.

Phase B Readiness

FUTURE OPPORTUNITY
10% — Phase B interfaces (Nerve Center, Concierge, Voice AI) require PHI; no skills today

All 12 Phase B interfaces require patient health information. Current Claude Skills are PHI-free by design, which aligns perfectly with Phase A. Phase B skills would need to integrate with PMS data post-BAA — a fundamentally different architecture. The HIPAA PHI Scrubber skill is the bridge.
