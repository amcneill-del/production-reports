The Honest Picture

What we have, what the Full Plan describes, and the real relationship between them.

What We've Built Working Today

15 Claude-powered automation skills that generate real, actionable intelligence — website audits, GBP analysis, content strategies, performance scorecards, scheduling scans, and more. These run on demand and produce polished deliverables (HTML dashboards, Word reports, Excel files). They are proven, repeatable, and in active use across field operations.

What the Full Plan Describes Proposal Stage

A 42-interface enterprise software platform with a 24-week build timeline, 15–20 person engineering team, AWS infrastructure (ECS, Cognito, MSK, Temporal), PHI/HIPAA compliance gates, and persistent data across 260+ practices. It's a serious architecture document from Citizen & Partners / bilt.studio — but it has not entered active development.

The key insight: These are fundamentally different categories of thing. Our skills are prompt-driven automation workflows that generate point-in-time intelligence. The Full Plan is a persistent enterprise application. One doesn't simply "become" the other — but they can be powerfully complementary. Our skills are the only part of this vision that actually works today.

Gap Analysis

The honest difference between what skills can do and what the Full Plan requires.

What the ChatGPT conversation got wrong: It claimed you've built "70–80% of WS2" and suggested adding Airtable + triggers to close the gap. That undersells the Full Plan and oversells the skills. The Full Plan's WS2 alone includes a Capture Bot PWA with SMS, a Studio Request Portal, a 58-second Brand Review approval flow, an Asset Registry on S3/CloudFront, and platform-integrated distribution monitoring. These are persistent application features — not things you can approximate with a spreadsheet layer.

Your skills generate intelligence. The Full Plan stores and acts on it. Core Gap

Every one of your skills follows the same pattern: Input → Analysis → Output file → Done. The Full Plan requires: Input → Stored in Practice Data Hub → Scored → Compared over time → Triggers generated → Actions assigned → Outcomes measured → Feedback loop.

This isn't a criticism — it's the nature of what these tools are. Claude skills are excellent at the "analysis → output" step. But they don't persist data, track changes longitudinally, or trigger automated workflows. That requires application infrastructure.

No persistent Practice Profile exists Infrastructure

The Full Plan's entire WS1 centers on a Practice Data Hub — a database seeded with 260 practices, CRUD API, RBAC, and event bus. Every interface reads from and writes to this hub. Your skills each generate standalone reports with no shared data layer. Running a Website Audit today doesn't inform next week's Marketing Strategy unless someone manually connects them.

No automation or trigger logic Orchestration

The Full Plan's Intervention Queue uses AI-generated composite risk scores to automatically surface practices that need attention and assign actions. Your skills require a human to decide which skill to run, for which practice, and when. The "trigger logic" suggested in the ChatGPT conversation (if score < 70 → run skill) sounds simple but actually requires persistent scoring + monitoring infrastructure to implement.

No vendor API integrations Data

The Full Plan pulls live data from Cinnamon (phone scoring), Swell (reviews), Dental Intel (production/clinical), and NexHealth (scheduling) via API integrations. Your skills work from manually uploaded CSVs or by scanning public-facing websites. Both approaches produce useful analysis, but the platform approach enables continuous monitoring while yours provides snapshots.

WS3 Culture OS has minimal coverage Coverage

WS3 is the Full Plan's cleanest workstream — all 8 interfaces ship in Phase A. It includes Morning Huddle displays, Champion Hub, CCA Academy (Bronze–Gold certification), Recognition & Rewards, and a Culture Admin Console. Your performance reporting tools (Marketing Performance, Referral Lab) touch the scoring/measurement aspect but don't address the behavior change, training, or daily workflow components.

Strategic Positioning

How to frame what you've built — without overclaiming or underselling.

Don't Say This

"We've already built 70–80% of the Content Engine workstream."

"We just need Airtable to connect everything."

"Our skills are early versions of the platform interfaces."

These overclaim the relationship and will erode credibility with anyone who reads the Full Plan closely.

Say This Instead

"We have working field intelligence tools that deliver value to Market Managers today — right now — while the platform is still in planning."

"Our skills prove the use cases the Full Plan is designed to systematize."

"Every skill we've built is a validated requirement for the platform. We're not guessing what the field needs — we've already built it and tested it."

This is honest, strong, and positions you as the person who validated the demand.

Your real strategic edge: The Full Plan is a proposal. Your skills are working software. In any organization, the person who has already built the thing that works has more leverage than the person with the best blueprint. Your skills are proof that the use cases in WS1, WS2, and WS3 are real — because people are already using them.

The Three Roles Your Skills Play

1. Immediate Value

Market Managers and field teams get actionable intelligence today without waiting for a 24-week platform build. This is the "what works right now" story.

2. Requirements Validation

Every skill you've built and tested is a proven use case. When the platform gets built, you've already validated what the field actually needs vs. what looks good on a spec sheet.

3. Future Data Source

When the Practice Data Hub exists, your skills can pipe their outputs into it. Website Audit scores become the Storefront Health metric. Content Engine calendars feed the Editorial Queue. The skills become input layers for the platform.

Realistic Roadmap

What you can actually do in the next 90 days — without pretending you're building a platform.

Now → 30 Days

Standardize & Document

Make what you have legible to the rest of the organization.

  • Create a skill catalog with clear descriptions, inputs, outputs, and use cases for each of the 15 skills
  • Define a consistent scoring framework (Access, Experience, Content, Growth) that every diagnostic skill maps to
  • Run 5–10 practices through a "full stack" workflow (Website Audit → GBP Audit → Content Engine → Strategy → Action Plan) to prove the end-to-end value chain
  • Document the results as case studies for leadership
30 → 60 Days

Build the "Practice Snapshot"

Not a database — a repeatable process that creates a composite view of any practice.

  • Design a standard Practice Snapshot template (HTML or PPTX) that aggregates outputs from Website Audit + GBP Audit + Scheduling Scan + Content Scout into a single view
  • Create a skill (or workflow) that chains these together and produces the snapshot automatically
  • Pilot with Market Managers on 20 practices to test utility and adoption
  • This is your "Practice Deep Dive lite" — a working version of what WS1 envisions, delivered through skills instead of infrastructure
60 → 90 Days

Prove the Prioritization Logic

Show that AI-driven prioritization works — even without an Intervention Queue.

  • Run Practice Snapshots across 50+ locations and rank them by composite score
  • Generate a "Weekly Priority Brief" that surfaces the bottom 10 practices and recommends specific skill-driven actions for each
  • Track whether the recommended actions were taken and what changed — this creates the feedback data the platform will eventually need
  • Present results to leadership as proof-of-concept for the Full Plan's Intervention Queue
90+ Days — Decision Point

Platform Build vs. Skill Scaling

By now you'll have the data to make the real strategic call.

  • If the Full Plan gets greenlit: Your skills become the requirements spec and your field data becomes the baseline. You've de-risked the build by proving demand.
  • If the Full Plan stays on hold: You've built a working field intelligence operation that delivers real value. Continue scaling skills, add lightweight persistence (shared spreadsheet, Airtable), and keep compounding.
  • Either way, you win — because you built the thing that works while the plan was still a proposal.
